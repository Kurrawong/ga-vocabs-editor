# About K-Maker

K-Maker is a vocabulary authoring and publishing platform built for organisations that manage controlled vocabularies. It provides a complete workflow for creating, editing, reviewing, and publishing SKOS vocabularies as static websites.

## What It Does

- **Browse** published vocabularies with hierarchical navigation and full-text search
- **Edit** concepts directly in the browser with an inline editor
- **Review** proposed changes through a structured staging workflow
- **Publish** approved vocabularies as static sites — no server infrastructure required
- **Share** vocabularies in multiple export formats (Turtle, JSON-LD, RDF/XML, CSV)
- **Embed** interactive vocabulary components in any web page

## How It Works

K-Maker takes vocabulary source files written in SKOS (Simple Knowledge Organization System) and generates a complete browsable website. The build pipeline processes Turtle (TTL) files into optimised JSON, builds a search index, and produces a static site that can be deployed anywhere.

Vocabulary authoring follows a structured pipeline: authors edit in their own workspace, submit changes for review in a shared staging area, and approved updates are promoted to the published site. This ensures quality control without slowing down the editing process.

## Standards

K-Maker is built on open standards:

- **SKOS** — Simple Knowledge Organization System for vocabulary structure
- **SHACL** — Shapes Constraint Language for validation and rendering profiles
- **DCAT** — Data Catalog Vocabulary for metadata
- **Dublin Core** — Standard metadata terms
