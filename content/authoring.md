# Authoring Vocabularies

K-Maker provides a structured workflow for editing and publishing vocabularies. Changes move through a clear pipeline from draft to production, with review at each stage.

## How It Works

The authoring workflow has three stages:

### 1. Edit

Sign in and select a vocabulary to begin editing. You can add, modify, or remove concepts directly in the browser. Changes are saved to your personal editing area — they won't affect the published vocabularies until you choose to promote them.

- Add new concepts with labels, definitions, and relationships
- Edit existing concept properties inline
- Reorder and restructure the concept hierarchy
- Changes are saved automatically as you work

### 2. Stage

When your edits are ready for review, push them to the **staging area**. This is a shared space where reviewers can see proposed changes before they go live.

- Submit your changes for review with a description of what was changed
- Reviewers can inspect the proposed updates
- Feedback can be incorporated before approval
- Multiple vocabulary updates can be staged together for a coordinated release

### 3. Publish

Once changes are approved in staging, they are promoted to **production**. This updates the published vocabulary that everyone sees.

- Approved changes are merged into the published vocabulary
- The site rebuilds automatically with the updated data
- Previous versions are preserved in the change history
- Published vocabularies are immediately available for download and use

## Getting Started

1. Click **Sign in** in the header to authenticate
2. Select a vocabulary from the [home page](/vocabs)
3. Enter edit mode to begin making changes
4. Save your work and submit for review when ready

The workflow ensures that vocabulary changes are always reviewed before publication, maintaining data quality across the organisation.
